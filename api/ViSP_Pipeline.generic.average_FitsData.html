average_FitsData¶

ViSP_Pipeline.generic.average_FitsData(fitsData, errData=None, preserve_modulator=True)[source]¶

Average a Data.FitsData object

In the most general case this will simply collapse along the last axis of the stacked data (i.e., collapse all hdus). However, there is also the option to preserve ViSP modulator states, in which case each state of the modulation cycle is collapsed separately.

In the first case the header information is taken from the first data hdu, and in the second case the header for each modulation state is taken from the first data hdu at that position.

If an uncertainty data set is provided then it will be propagated through the averaging process (i.e., a quadrature sum)

Parameters
fitsDataData.FitsData

Should probably actually have some data in it

errData: Data.FitsData

An optional data object containing pixel uncertainty estimates. Should be exactly the same size as fitsData

preserve_modulatorbool

If True then keep the separate modulation states separate during the average

Returns
Data.FitsData

The averaged data

Data.FitsData (optional)

If errData was specified then an uncertainty estimate of the average is also returned
