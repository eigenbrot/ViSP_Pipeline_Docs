dark_correction¶

ViSP_Pipeline.generic.dark_correction(fitsData, dark_calibration, errData=None, dark_calibration_err=None)[source]¶

Subtract dark data from every HDU (probably a modulation state) in the input data.

If uncertainty estimates are provided then the uncertainty on the input data will be updated to be the quadrature sum of itself and the provided dark uncertainty.

Parameters:

fitsData : Data.FitsData

Data object to perform dark subtraction on

dark_calibration : Data.FitsData

Object containing a processed dark calibration. If, for some reason, there are more than 1 data HDU, only the first data HDU is used.

errData : Data.FitsData

The pixel uncertainty estimates of the input data

dark_calibration_err : Data.FitsData

The pixel uncertainty estimates of the dark calibration data
Returns:

None

The data (and uncertainty, if provided) are updated in place
