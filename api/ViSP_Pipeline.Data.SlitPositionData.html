SlitPositionData¶

class ViSP_Pipeline.Data.SlitPositionData(data_dir=None, dataset_ID='', spatial_step=0, suffix='', memmap=None)[source]¶

Bases: ViSP_Pipeline.Data.FitsData

SlitPositionData is an extension of FitsData that loads a single ViSP slit position (i.e. spatial step) from a directory containing a full ViSP DataSet. The user is able to specify which spatial step will be loaded and after loading this object behaves exactly like FitsData.

Initialize the object and maybe load some data.

Parameters
data_dirstr

Path to directory in which to look for data

dataset_IDstr

Only load data from this data set. A check is made against each file’s ID___004 header value. Can be a blank string to just load all data sets.

spatial_stepint

Only files/data pertaining to this spatial step will be loaded

suffixstr

Optional file mask. The full mask is DATA_DIR/*SUFFIX. Defaults to “.fits”

memmapbool

The decision on whether or not to use astropy.io.fits’ memmap feature. If set to ‘None’ then astropy will figure it out for you (recommended).

load_from_directory(data_dir, dataset_ID='', suffix='')[source]¶

Load DHS-style data from a directory.

It is assumed that all relevant files in the directory have SPEC-0122 headers and a single HDU. In other words, each file is exactly one SysOutput. Only files whose InstrumentProgramTask (DKIST004) is ‘Observation’ and whose spatial step (VISP_CSS) is self.spatial_step will be loaded.

Parameters
data_dirstr

Path to the directory containing data to load

dataset_IDstr

Only files whose ID___004 header value matches this parameter will be loaded. Can be a blank string to load all data sets.

suffixstr

Optional file mask. The full mask is DATA_DIR/*SUFFIX. Defaults to “.fits”

load_from_filelist(file_list)[source]¶

Load files from a list of file names

Each file is checked to make sure it is from the correct spatial step (the one that this class was initialized with), but no checks are made against program task or dataset ID.

Parameters
file_listlst

A list of paths to files to load
