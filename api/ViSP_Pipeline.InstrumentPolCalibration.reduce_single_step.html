reduce_single_step¶

ViSP_Pipeline.InstrumentPolCalibration.reduce_single_step(pol_dir, CS_step, outpre, dark_cal=None, solar_gain_cal=None, geometric_cal=None, dark_cal_err=None, solar_gain_cal_err=None, gen_err=False, cu_numbins_x=1, cu_numbins_y=1, demod_numbins_x=1, demod_numbins_y=1, dark_lock=None, gain_lock=None, geo_lock=None)[source]¶

Perform full ViSP reduction on a single step from a PolCal Calibration Sequence (CS)

Dark, gain, and geometric effects are removed and the two beams are split into two separate objects/files. An arbitrary amount of binning of the final data is also possible. Binning the data results in faster PA&C analysis, at the expense of spatial/spectral resolution in the derived Calibration Unit parameters.

Parameters:

pol_dir : str

Path to the directory containing a single SoCC

CS_step : int

Calibration Sequence step number to use for reduction

outpre : str

Prefix of the location to save processed data. Can include the full path. The processed data will be saved as outpre_beam1.fits and outpre_beam2.fits, and associated uncertainties (if provided) will be saved as outpre_beam1_err.fits and outpre_beam2_err.fits.

dark_cal : Data.FitsData

A Dark Calibration object. Probably the result of DarkCalibration.main().

solar_gain_cal : Data.FitsData

A Solar Gain Calibration object. Probably the result of SolarGainCalibration.main().

geometric_cal : Data.FitsData

A Geometric Calibration object. Probably produced by SolarGainCalibration.main().

dark_cal_err : Data.FitsData

Uncertainty associated with dark_cal

solar_gain_cal_err : Data.FitsData

Uncertainty associated with solar_gain_cal

gen_err : bool

If True then pixel uncertainty estimates will be generated from the raw data frames

cu_numbins_x : int

Each beam will have this number of spatial bins when computing the best CU parameters

cu_numbins_y : int

Each beam will have this number of spectral bins when computing the best CU parameters

demod_numbins_x : int

Each beam will have this number of spatial bins in the final demodulation matrices

demod_numbins_y : int

Each beam will have this number of spectral bins in the final demodulation matrices

dark_lock : multiprocessing.synchronize.lock

This lock will be used when reading information from the dark calibration object. This avoids some weirdness in astropy’s FITS containers when accessing the same data at the same time.

gain_lock : multiprocessing.synchronize.lock

This lock will be used when reading information from the gain calibration object. This avoids some weirdness in astropy’s FITS containers when accessing the same data at the same time.

geo_lock : multiprocessing.synchronize.lock

This lock will be used when reading information from the geometric calibration object. This avoids some weirdness in astropy’s FITS containers when accessing the same data at the same time.
Returns:

Data.FitsData

PolCal ViSP data for beam 1

Data.FitsData

PolCal ViSP data for beam 2

Data.FitsData

If uncertainty on the input data is provided, the uncertainty associated with beam 1 PolCal data

Data.FitsData

If uncertainty on the input data is provided, the uncertainty associated with beam 2 PolCal data
