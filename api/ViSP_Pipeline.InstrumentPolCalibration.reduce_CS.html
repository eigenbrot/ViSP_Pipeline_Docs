reduce_CS¶

ViSP_Pipeline.InstrumentPolCalibration.reduce_CS(raw_pol_dir, raw_dark_dir, raw_lamp_gain_dir, raw_solar_gain_dir, cu_numbins_x=1, cu_numbins_y=1, demod_numbins_x=1, demod_numbins_y=1, dark_cal='DarkCal', lamp_gain_cal='LampGainCal', solar_gain_cal='SolarGainCal', geometric_cal='GeoCal', threads=1, gen_noise_frames=False)[source]¶

ViSP reduce all steps in a set of PolCal Calibration Sequence (CS) observations

This function basically constructs the correct config and kwarg objects and then does all of the actual processing via calls to generate_intermediate_objects and manage_CS_rdx.

Parameters
raw_pol_dirstr

Location of directory that contains raw Calibration Sequence (CS) data

raw_dark_dirstr

Location of the directory containing raw dark frames associated with the CS

raw_lamp_gain_dirstr

Location of the directory containing raw lamp gain frames associated with the CS

raw_solar_gain_dirstr

Location of the directory containing raw solar gain frames associated with the CS

cu_numbins_xint

Each beam will have this number of spatial bins when computing the best CU parameters

cu_numbins_yint

Each beam will have this number of spectral bins when computing the best CU parameters

demod_numbins_xint

Each beam will have this number of spatial bins in the final demodulation matrices

demod_numbins_yint

Each beam will have this number of spectral bins in the final demodulation matrices

dark_calstr

Location of the processed DarkCalibration object to either load or save

lamp_gain_calstr

Location of the processed LampGainCalibration object to either load or save

solar_gain_calstr

Location of the processed SolarGainCalibration object to either load or save

geometric_calstr

Location of the processed GeometricCalibration object to either load or save

threadsint

Maximum number of cpus/threads to use during reduction

gen_noise_framesbool

If True then uncertainty estimates will be generated for the reduced CS step data

Returns
None
