main¶

ViSP_Pipeline.InstrumentPolCalibration.main(raw_pol_dir, output, raw_dark_dir, raw_lamp_gain_dir, raw_solar_gain_dir, gen_noise_frames=False, telescope_db=None, threads=1, cu_numbins_x=1, cu_numbins_y=1, demod_numbins_x=1, demod_numbins_y=1)[source]¶

Run raw data (i.e., a PolCal CS observation) through the Instrument Polarization Calibration pipeline

The steps in the pipeline are:

  1. Generate intermediate (dark, gain, etc.) Calibration objects associated with CS

  2. Perform ViSP reductions on entire CS data set

  3. Fit CU parameters and generate demodulation matrices from the resulting SoCCs

So basically, this pipeline calls the normal ViSP pipeline and then uses PA&C Modules to further analyze the reduced PolCal data.

Parameters
raw_pol_dirstr

Path to directory containing raw PolCal data

outputstr

Path were final InstrumentPolCalibration (i.e., demodulation matrices) will be saved

raw_dark_dirstr

Path to directory containing raw dark data associated with PolCal observations

raw_lamp_gain_dirstr

Path to directory containing raw lamp gain data associated with PolCal observations

raw_solar_gain_dirstr

Path to directory containing raw solar gain data associated with PolCal observations

gen_noise_framesbool

If True then uncertainty estimates will be generated for all Calibration and data products

telescope_dbstr

Path to the database of telescope properties (x, au). If None then the PA&C default will be used.

cu_numbins_xint

Each beam will have this number of spatial bins when computing the best CU parameters

cu_numbins_yint

Each beam will have this number of spectral bins when computing the best CU parameters

demod_numbins_xint

Each beam will have this number of spatial bins in the final demodulation matrices

demod_numbins_yint

Each beam will have this number of spectral bins in the final demodulation matrices

threadsint

The number of independent processes to use when performing ViSP reductions on the PolCal data

Returns
None
