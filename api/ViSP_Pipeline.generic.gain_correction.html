gain_correction¶

ViSP_Pipeline.generic.gain_correction(fitsData, gain_calibration, errData=None, gain_calibration_err=None)[source]¶

Divide gain data from from the input data. Modulation states are preserved.

If uncertainty estimates are provided then the uncertainty on the input data will be correctly updated to include the uncertainty on the gain calibration.

Parameters
fitsDataData.FitsData

Data object to perform gain division on

gain_calibrationData.FitsData

Object containing a processed gain calibration. If, for some reason, there are more than 1 data HDU per modulation state only the first data HDU is used.

errDataData.FitsData

The pixel uncertainty estimates of the input data

gain_calibration_errData.FitsData

The pixel uncertainty estimates of the gain calibration data

Returns
None

The data (and uncertainty, if provided) are updated in place
