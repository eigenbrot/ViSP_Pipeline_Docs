average_beams¶

ViSP_Pipeline.ScienceCalibration.average_beams(fitsData, errData=None, preserve_modulator=False)[source]¶

Average the two beams together. Real simple.

This function is modulator state aware, although I’m not sure why.

Parameters
fitsDataData.FitsData

Data containing two beams across N modulator states

errDataData.FitsData

Uncertainty on fitsData

preserve_modulatorbool

If True then only beams from the same modulator state will be averaged together. Otherwise, average all beams together.

Returns
Data.FitsData

The average of both beams. Depending on the value of preserve_modulator there will either be a single HDU for each modulator state, or just one HDU containing the global average.

Data.FitsData or None

If errData is provided then this will be the uncertainty associated with the output
