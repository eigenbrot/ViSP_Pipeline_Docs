extract_macropixels¶

ViSP_Pipeline.InstrumentPolCalibration.extract_macropixels(fitsData, numpix_x, numpix_y, errData=None)[source]¶

Bin the input data down to a specified number of pixels.

Because this function is designed to produce PolCal data the two ViSP beams are split apart and placed into separate Data.FitsData objects.

If the number of output pixels does not divide evenly into the number of input pixels then some input pixels will be lost. This might or might not be a bad thing.

Parameters:

fitsData : Data.FitsData

Object containing data to bin. Should contain both ViSP beams and is probably the output of ScienceCalibration.geometric_correction()

numpix_x : int

The number of spatial output pixels. Both beams will have the same number of spatial pixels.

numpix_y : int

The number of spectral output pixels PER BEAM

errData : Data.FitsData

Uncertainty associated with the input data
Returns:

Data.FitsData

PolCal ViSP data for beam 1. Each modulator state in a separate ImageHDU.

Data.FitsData

PolCal ViSP data for beam 2. Each modulator state in a separate ImageHDU.

Data.FitsData

If uncertainty on the input data is provided, the uncertainty associated with beam 1 PolCal data

Data.FitsData

If uncertainty on the input data is provided, the uncertainty associated with beam 2 PolCal data
