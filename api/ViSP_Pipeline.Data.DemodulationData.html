DemodulationData¶

class ViSP_Pipeline.Data.DemodulationData(array=None, header=None, outputfile=None, overwrite=True)[source]¶

Bases: ViSP_Pipeline.Data.FitsData

DemodulationData is a small modification of the FitsData class. It requires that input arrays have 4 dimensions and provides a ‘data’ property to access the demodulation matrix.

Initialize the object and generate an HDU list with the input array

Parameters:

array : numpy.ndarray

A set of demodulation matrices. Must be 4 dimensional.

header : astropy.io.fits.header.Header

If provided, all HDUs in the current object will be given this header.

outputfile : str

If provided, this object’s HDUList will be written to this file.

overwrite : bool

If True then outputfile will be overwritten if present.

init_with_ndarray(array, outputfile=None, overwrite=True)[source]¶

Load data from a demodulation-style numpy.ndarray and construct an astropy.io.fits HDU list

Because all demodulation matrices should have the same shape (X, Y, 4, M), a check is made to ensure this is true.

If an output file is not specified then a temporary FITS file will be created to take advantage of astropy.io.fits’ memmap ability. In this case the FitsData object will be completely lost when it is no longer needed. To save the object use the .write_out() method prior to deletion.

Parameters:

array : numpy.ndarray

Array to be loaded into the FitsData object. Must have exactly 4 dimensions.

outputfile : str

If specified, a FITS file will be saved to this path and then reloaded. If None then a temporary file will be created.

overwrite : bool

If True the outputfile specified will be overwritten.
