DemodulationData¶

class ViSP_Pipeline.Data.DemodulationData[source]¶

Bases: ViSP_Pipeline.Data.FitsData

DemodulationData is a small modification of the FitsData class. It keeps track of demodulation matrices for each beam and allows the beams to either have shape (X, Y, 4, M) or (4, M).

init_beam_with_ndarray(array, beam, header=None)[source]¶

Load data from a demodulation-style numpy.ndarray and construct an astropy.io.fits HDU list

Because all demodulation matrices should have shape (X, Y, 4, M) or (4, M), a check is made to ensure this is true.

A temporary FITS file will be created to take advantage of astropy.io.fits’ memmap ability. Thie means this object will be completely lost when it is no longer needed. To save the object use the .write_out() method prior to deletion.

Parameters:

array : numpy.ndarray

Array to be loaded into the FitsData object. Must have exactly 4 dimensions.

beam : int

The ViSP beam that the specified demodulation matrix belongs to
