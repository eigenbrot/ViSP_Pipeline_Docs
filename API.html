ViSP_Pipeline.Data Module¶

Classes¶

AlignmentData([data_dir, memmap])

AlignmentData is an extension of FitData designed to only hold data associated with Alignment. It inherits all of the functionality of FitsData and defines an extra method that loads all data from a directory who’s headers match the following: Key Value — —– DKIST004 Alignment.

CSStep(header)

This class allows for an easy way to quickly compare SPEC-0122 style headers to determine if they come from the same Calibration Sequence (CS) step.

CalibrationCurveData([CS_step, memmap])

CalibrationCurveData is an extension of FitsData that loads a single Calibration Curve from a directory containing a full Set of Calibration Curves (SoCC).

DarkData([data_dir, memmap])

DarkData is an extension of FitData designed to only hold data associated with Dark. It inherits all of the functionality of FitsData and defines an extra method that loads all data from a directory who’s headers match the following: Key Value — —– DKIST004 Dark.

DataSetData([data_dir, dataset_ID, suffix])

DataSetData is designed to handle the aggregation of a bunch of files that belong to a single ViSP data set.

DemodulationData()

DemodulationData is a small modification of the FitsData class.

FitsData([fitsfile, memmap, ignore_shape_check])

FitsData is a basically a wrapper for astropy.io.fits, with some extra methods and properties to aid the reduction of DKIST data.

LampGainData([data_dir, memmap])

LampGainData is an extension of FitData designed to only hold data associated with LampGain. It inherits all of the functionality of FitsData and defines an extra method that loads all data from a directory who’s headers match the following: Key Value — —– DKIST004 Gain PAC__002 Lamp PAC__003 On.

SoCCData performs 2 functions: First, it separates and sorts Calibration Sequence (CS) steps found among raw data in a directory.

SolarGainData([data_dir, memmap])

SolarGainData is an extension of FitData designed to only hold data associated with SolarGain. It inherits all of the functionality of FitsData and defines an extra method that loads all data from a directory who’s headers match the following: Key Value — —– DKIST004 Gain PAC__002 Clear TELSCAN [‘Raster’, ‘Random’, ‘Spiral’].
